# CF7 Validate Pro — Design Spec
**Date:** 2026-04-01
**Plugin:** `cf7-validate-pro`
**Author:** Shon
**Status:** Approved

---

## Overview

**CF7 Validate Pro** — a standalone WordPress addon plugin by Shon for Contact Form 7 (CF7) that adds comprehensive client-side (JS) and server-side (PHP) validation to any CF7 form. Validation rules are configured per-form via a custom **Validation tab** added to the CF7 form editor (built with React using `@wordpress/scripts`). Sensible defaults are auto-applied based on field type; the admin can override any rule per form.

---

## Architecture

### Approach
Independent validation layer running on top of CF7. The plugin hooks into CF7 at defined intercept points (form render, AJAX response, before send mail) without modifying CF7 core files. PHP and JS share the same rule set, both reading from the same config stored as CF7 form post meta.

### Bootstrap Flow
1. On activation — check CF7 is active; if not, deactivate self and show admin notice
2. On each page load — enqueue JS/CSS only on pages containing a CF7 form shortcode
3. Validation config loaded from post meta, merged with defaults, passed to JS via `wp_localize_script` keyed by form ID
4. Multiple forms on same page — on page load, JS scans all CF7 form elements and assigns a sequential `instance_index` (0, 1, 2…) per occurrence. Each instance gets a runtime key (`{form_id}_{instance_index}`) so validation state, counters, and intl-tel-input instances are fully isolated even when the same form appears more than once

## Configuration — Validation Tab

### UI
A **Validation** tab is added to the CF7 form editor using CF7's panel registration API. Built with React (`@wordpress/scripts`). The tab:
- Parses the current form's field tags to list all fields
- Shows contextual rule inputs per field based on detected field type
- Has a **Global Settings** section at the top

### Global Settings
| Setting | Type | Default |
|---|---|---|
| Show "Optional" label on non-required fields | Toggle (on/off) | Off |

### Per-Field Settings (contextual by type)

**All field types:**
| Setting | Type |
|---|---|
| Field Label (for error messages) | Text input (auto-generated from field name if blank — algorithm: strip leading `your-`/`the-`, replace `-`/`_` with spaces, title case. e.g. `your-first-name` → `First Name`) |
| Required | Toggle |

**Text fields:**
| Setting | Type | Default |
|---|---|---|
| Min characters | Number | — |
| Max characters | Number | — |
| Alpha only | Toggle | Off |
| No leading spaces | Toggle | Off |
| Allow special characters | Toggle | On |
| Allow emoji/Unicode | Toggle | On |
| Collapse consecutive whitespace | Toggle | On |
| Input mask pattern | Text (`9`=digit, `a`=letter, `*`=alphanumeric) | — |
| Counter format | Select: `count/max` / `remaining` / `off` | `count/max` |

**Name fields (first-name, last-name — auto-detected):**
Defaults applied automatically:
- Alpha only: On
- No leading spaces: On
- First name min: 2, max: 56
- Last name max: 56 (no min)

**Email fields:**
- Trim leading/trailing whitespace: always on
- Format validation: always on

**Numeric fields:**
| Setting | Type | Default |
|---|---|---|
| Min value | Number | — |
| Max value | Number | — |
| Allow negative values | Toggle | Off |
| Allow zero | Toggle | On |

**Phone fields:**
| Setting | Type | Default |
|---|---|---|
| Min length | Number | 7 |
| Max length | Number | 15 |
| Enable international input (intl-tel-input) | Toggle | Off |
| Default country | Select | Auto (browser locale) |

**Textarea / Message fields:**
| Setting | Type | Default |
|---|---|---|
| Max characters | Number | 1500 |
| Counter format | Select: `count/max` / `remaining` / `off` | `count/max` |
| Max height (px) for scrollbar | Number | 200 |
| Security sanitization (strip code operators) | Toggle | On |

**Select / Dropdown fields:**
| Setting | Type | Default |
|---|---|---|
| Required | Toggle | Off |
| Default/placeholder option value | Text | `""` |

**Checkbox / Radio fields:**
| Setting | Type | Default |
|---|---|---|
| Required (at least one) | Toggle | Off |
| Default state | Toggle per option | — |

**File upload fields:**
| Setting | Type | Default |
|---|---|---|

| Allow multiple files | Toggle | Off |
| Show file preview | Toggle | Off |

### Config Storage
All settings saved as post meta on the CF7 form post (`_cfv_validation_config`), serialized as JSON.

### Form Duplication
When a CF7 form is duplicated, the plugin hooks into `wp_insert_post` and checks if the new post was created as a copy of an existing CF7 form (detected via common duplication plugins' meta flags, e.g. `_dp_original` from Duplicate Post). If so, it copies `_cfv_validation_config` from the source form to the new form post automatically.

## Client-Side Validation (JS)

### Event Triggers
Validation runs on: `focus`, `input` (change), `blur`, and form `submit`.

### Per-Form Isolation
On page load, each CF7 form instance is identified by `{form_id}_{instance_index}`. All state (errors, counter values, intl-tel-input instances) is stored per instance key. The same form appearing twice on a page runs two fully independent validators.

### Validation Rules

| Rule | Implementation |
|---|---|
| Required | Block if empty after `trim()` |
| Alpha only | Reject non-letter characters (spaces between words allowed, not leading) |
| No leading spaces | Reject/warn if value starts with whitespace |
| Min/max length | `trimmedValue.length` check |
| Email format | Regex + trim leading/trailing whitespace |
| Phone (standard) | Strip allowed chars (`+`, `-`, digits, spaces, `()`), length check |
| Phone (intl-tel-input) | Read `iti.getNumber()` for all length/format checks; skip special char stripping on visible field |
| URL | Regex check for `http://` or `https://` prefix + basic URL pattern |
| Numeric only | Block non-digit characters on `input` event |
| Numeric min/max | Compare parsed float against configured limits |
| Negative/zero check | Reject based on config |
| Special characters | Strip or reject based on config |
| Emoji/Unicode | Strip non-ASCII if disabled |
| Consecutive whitespace | Collapse on `blur` |
| Input mask | Lightweight mask applied on `focus`/`input` using `9`/`a`/`*` pattern; raw value (mask stripped) used for server submission |
| File type | Check extension against allowed list on `change` |
| File size | Check `file.size` against max bytes on `change` |
| Security (message) | Live warning shown if code-like patterns detected; not blocked client-side |
| Select required | Check selected value !== default/placeholder value |
| Checkbox/radio required | Check at least one in group is checked |

### Error Display
- Each field has a `<span class="cfv-error-tip" aria-live="polite">` injected directly below it on page load (empty, hidden)
- On failure: span populated and shown inline next to the field
- On correction: span cleared and hidden
- CF7's native `.wpcf7-not-valid-tip` spans hidden via CSS (`display: none !important`)
- Errors for checkbox/radio groups shown next to the group wrapper

### Character Counter
- Injected below applicable fields (textarea, text fields with max length)
- Updates live on `input` event
- Turns red when within 10% of limit
- Turns red + bold at/over limit; blocks form submit

### Submit Button
- On submit click: disabled + CSS spinner shown
- If client validation fails: spinner removed, button re-enabled, focus moved to first error field
- On successful CF7 response: all fields reset to default, errors cleared, button re-enabled
- intl-tel-input fields reset via `iti.setNumber('')` and country reset to configured default

## Server-Side Validation (PHP)

### Hook Point
`wpcf7_before_send_mail` filter — rejects submission with `validation_failed` status if any rule fails. CF7 never sends the email on failure.

### Rules
Same ruleset as client-side. PHP implementations:

| Rule | PHP |
|---|---|
| Required | `trim()` + empty check |
| Alpha only | `preg_match('/^[a-zA-Z\s]+$/')` |
| No leading spaces | `ltrim()` comparison |
| Min/max length | `mb_strlen(trim($value))` |
| Email | `filter_var(FILTER_VALIDATE_EMAIL)` + trim |
| Phone (standard) | Strip allowed chars, length check |
| Phone (intl-tel-input) | Read `$_POST['cfv_phone_full_{field_name}']`; validate E.164 format (`+` + 8–15 digits) |
| URL | `filter_var(FILTER_VALIDATE_URL)` + scheme check |
| Numeric | `is_numeric()` + range checks |
| File type | Check `$_FILES` extension (not MIME — MIME can be spoofed) |
| File size | Check `$_FILES['size']` against max bytes |
| Security sanitization | `strip_tags()`, strip `<script>`, SQL operators (`--`, `;`, `DROP`, `INSERT`, `SELECT`), `eval(`, `alert(`, `<?`, `?>`, `{`, `}` |
| Select required | Check submitted value !== placeholder value |
| Checkbox/radio required | Check at least one value present in submission |

### Error Response
Failures return CF7's `validation_failed` status. Field-specific messages injected into CF7's AJAX response JSON via `wpcf7_ajax_json_echo` filter — same error UI renders for server-side failures as for client-side.

### Phone Full Number Submission
When intl-tel-input is enabled:
- JS writes full E.164 number to hidden input `cfv_phone_full_{field_name}` before submit
- PHP reads this hidden field and overrides the CF7 submission data for that field
- Full number (e.g., `+447911123456`) appears in the CF7 email, not the local digits

## UX Components

### Required Field Asterisks
- Plugin hooks into `wpcf7_form_elements` to post-process rendered form HTML
- Injects `<span class="cfv-required-asterisk">*</span>` next to label of each required field
- CSS: red color

### Optional Field Label
- Controlled by global toggle (per form)
- When ON: `(Optional)` injected next to label of all non-required fields
- Styled: muted grey, smaller font
- When OFF: no optional labels anywhere on the form

### Inline Error Messages
- Format: `{Field Label} is required` / `{Field Label} must be at least 2 characters` / etc.
- All errors appear directly below their respective input field
- No summary block

### Error Message Reference

| Scenario | Message |
|---|---|
| Required (any field) | `{Field Label} is required` |
| Alpha only | `{Field Label} must contain letters only` |
| Leading space | `{Field Label} must not start with a space` |
| Min length | `{Field Label} must be at least {min} characters` |
| Max length | `{Field Label} must be no more than {max} characters` |
| Email invalid | `{Field Label} must be a valid email address` |
| Phone invalid | `{Field Label} must be a valid phone number` |
| URL invalid | `{Field Label} must be a valid URL including http:// or https://` |
| Numeric only | `{Field Label} must contain numbers only` |
| Below min value | `{Field Label} must be at least {min}` |
| Above max value | `{Field Label} must be no more than {max}` |
| Negative not allowed | `{Field Label} must be a positive number` |
| Zero not allowed | `{Field Label} cannot be zero` |
| File type | `{Field Label} accepts {types} files only` |
| File size | `{Field Label} must be under {max}MB` |
| Select required | `{Field Label} is required` |
| Checkbox/radio required | `{Field Label} — please select at least one option` |
| Security chars detected | `{Field Label} contains invalid characters` |

### Message Field Scrollbar
- Textarea gets `overflow-y: auto` and configurable `max-height` (default `200px`)
- Scrollbar appears as content exceeds max-height
- Works alongside character counter

### Checkbox / Radio UX
- Disabled options: CSS class `cfv-disabled`, visually greyed out, JS blocks interaction
- Label click toggles checkbox — preserved via standard `<label for="">` HTML
- Error shown next to group wrapper, not individual options
- Default state restored on form reset

## International Phone Input (intl-tel-input)

- Library loaded from plugin assets — no CDN
- Enqueued only on pages where at least one phone field has intl-tel-input enabled
- Each intl-tel-input instance keyed by `{form_id}_{instance_index}_{field_name}`
- On form reset: `iti.setNumber('')` called, country reset to configured default
- Conflict resolution with Section 3 validation:
  1. Length/format checks use `iti.getNumber()`, not raw field value
  2. Special character stripping skipped on visible field when intl-tel-input active
  3. All event handlers (`focus`, `input`, `blur`, `submit`) read `iti.getNumber()` when available

## Security

- File uploads: extension checked server-side (not MIME type, which can be spoofed); filenames sanitized (strip special chars and spaces)
- Message field: server-side `strip_tags()` + pattern stripping; client-side live warning only
- Nonce verification on all AJAX requests (Validation tab save, config load)
- All config values sanitized on save via `sanitize_text_field()`, `absint()`, `wp_kses()` as appropriate
- No external CDN dependencies — all assets served from plugin

## Out of Scope

- CF7 CAPTCHA / spam protection — not modified, CF7 handles this
- CF7 email configuration — not modified
- Custom form styling beyond validation UX elements
- Automated test suite — QA checklist (`QA Checklist - Sheet1.csv`) serves as the test plan
